Why BJJ Needs a Writing System

Not all martial arts encode information the same way. The difference is structural — and it determines how the art should be taught, analysed, and computed.

Alphabetic

Striking Arts

Boxing, Muay Thai, Karate. In an alphabetic system, symbols represent individual sounds — meaning is built by sequencing letters. A jab is a letter. It has no meaning alone. Sequence it — jab, cross, hook — and you get a word.

Striking is generative: you combine atomic actions in time. A finite alphabet produces infinite sequences. The natural mathematical framework is control theory — trajectories through a state space of joint angles.

Analogy: English. 26 letters → infinite words.
Logographic

Brazilian Jiu-Jitsu

In a logographic system, symbols represent meaning units. A Chinese character is not a sequence of sounds — it is a whole idea, a morpheme. You don't read it left-to-right; you recognize its global structure.

BJJ works the same way. The smallest unit of meaning is not a movement — it is a position. Mount is a character. Guard is a character. You recognize them by their structural invariants, not by reading a sequence of moves. The natural framework is category theory — objects and morphisms.

Analogy: Chinese. Characters = positions. Radicals = invariants.

The Pedagogical Problem

Most BJJ instruction uses an alphabetic approach: memorise this sweep, drill that escape, chain these submissions. Students drown in unrelated techniques — strokes without characters.

The logographic model says: teach the characters first (positions), then the radicals (structural invariants that make a position what it is), then the strokes (grip details, angle refinements, micro-adjustments).

1
Positional Literacy

Can the student recognise and name the position? Mount, Guard, Back, Side, Turtle.

2
Structural Invariants

What makes Mount Mount? Bilateral leg contact around the torso, facing opposed, opponent grounded. Lose the invariant, lose the position.

3
Semantic Stability

High Mount, Low Mount, Grapevine Mount are the same character in different fonts. Seatbelt Back Control and Body Triangle Back Control preserve the same radical.

4
Transitions as Morphisms

A sweep is not a "move." It is a morphism — a structure-preserving map that transforms one radical into another by changing the constraint geometry.

5
Stroke Order (Last)

Whether your elbow is flared 30° or 40° is stroke order — vital for mastery, confusing for literacy. Teach it last, not first.

The Algebra: BLISP

BLISP encodes every grappling position as a composition of a single primitive — the CON relation — plus geometric frame constraints. No other relation types exist. GRP, HOOK, CLP, PIN are all derived macros. One primitive. Everything else is structure.

CON — The Primitive

A typed contact between two oriented body axes:

CON(attacker, axis, depth, helicity)

  attacker  —  the limb doing the wrapping
  axis      —  the limb being wrapped around
  depth     —  symbolic depth (d, d1, d2, ...)
  helicity  —  winding direction (+, -, or 0)

Both attacker and axis carry role (Me/Op), body part, laterality sign (+/−), and orientation (from → to endpoints).

Frame Constraints

Geometric predicates on the spatial configuration:

FacingOpposed()    —  y = -y′  (face to face)
FacingAligned()    —  y =  y′  (same direction)
OnGround(X)       —  Z0(X)   (part X grounded)
NotOnGround(X)    —  ¬Z0(X)  (part X elevated)
KneeBracket(X)    —  KBR(X)  (knees straddle X)
NotKneeBracket(X) —  ¬KBR(X)

Frames narrow the spatial context without adding new primitives. KBR is a derived geometric predicate over joint positions.

Algebraic Operations

Four operations transform CON tuples. Three are involutions — applying twice returns the original:

HEL-FLIP
Flip helicity, preserve everything else. Involution.
HEL-FLIP(CON(a, b, d, h)) → CON(a, b, d, ¬h)
AX-REV
Reverse axis orientation — swap from/to endpoints. Involution.
AX-REV(CON(a, AX(x,p,q), d, h)) → CON(a, AX(x,q,p), d, h)
AX-SUB
Substitute the target axis with a new one.
AX-SUB(CON(a, b, d, h), c) → CON(a, c, d, h)
POV-SWAP
Swap attacker/axis, flip helicity, rename Me↔Op. Involution. Changes the observer.
POV-SWAP(CON(a, b, d, h)) → CON(b′, a′, d, ¬h)

Example: DLR and SLX are HEL-FLIP Duals

De La Riva
CON(Me.Le−, Op.Le+, d, −)
HEL-FLIP ↔
Single-Leg X
CON(Me.Le−, Op.Le+, d, +)

Same attacker, same axis, same depth. Only the winding direction differs.

The Pipeline

Four stages convert raw video frames into symbolic position sequences. No neural network classifies the position — contacts are inferred from keypoint geometry and matched algebraically against the radical dictionary.

01

Pose Extraction

COCO 17-keypoint poses for both athletes. Virtual parts (torso, back, neck, shin) reconstructed from joint geometry.

02

Contact Inference

Candidate CON tuples generated by limb-axis proximity. Each carries confidence from distance and joint visibility.

03

Radical Matching

Bottleneck scoring: weakest required contact dominates via geometric mean. Strong single-contact beats weak multi-contact.

04

Temporal Filtering

Persistence (N=8) suppresses flicker. Closure memory carries intermittent Fo-Fo evidence across nearby frames.

Current Results

Recognition accuracy using purely algebraic matching — no neural network for classification. Contacts inferred from 2D keypoints, scored against the radical dictionary with bottleneck matching and temporal persistence.

No learned classifier. Every number above comes from purely symbolic matching. The system infers contacts from keypoint geometry and scores them algebraically.
Coarse ground truth. The ViCoS labels collapse all open-guard subtypes into one class. The algebra distinguishes DLR from SLX from RDLR — the metric measures whether the predicted radical falls within the correct position family.
2D projection limit. All inference runs on 2D keypoints from a single camera. Depth ambiguity is the main error source. 3D pose or multi-view would improve every number.

Design Principles

One Primitive

CON is the only relation. GRP, HOOK, CLP, PIN are derived macros over CON + frame constraints. No relation-type proliferation.

Algebraic Closure

HEL-FLIP, AX-REV, and POV-SWAP are involutions. Applying any twice returns the original. Self-consistent by construction.

Observer-Invariant

POV-SWAP transforms any position to the opponent's perspective. Mount from top becomes mount from bottom. Both views are algebraically encoded.

Explicit Radicals

Every position is a hand-written literal. No auto-generation, no learned embeddings. If you can read the radical, you understand the position.

Bottleneck Scoring

Match quality is dominated by the weakest required contact via geometric mean. Specificity is rewarded, not punished.

Temporal Coherence

Persistence + closure memory smooth predictions. Label flicker: 0.21 → 0.01 switches per frame.
